# 🚀 CSS Learning Journey

Welcome to my **CSS Course** practice repository! I am currently learning web development and using this project to practice everything from foundational concepts to layout positioning.

---
### 📂 About This Repository

This project is structured chronologically to reflect my learning process. To understand my coding approach, please note:

> [!NOTE]
> **Core Styling Approach:**  
> * **Phase 1 (Basics & Experiments):** Initially, I used **Internal CSS** within HTML files for rapid prototyping and quick conceptual testing.
> * **Phase 2 (Advanced & Projects):** As I progressed to complex layouts, I transitioned to **External CSS** to ensure clean, modular, and industry-standard code structures.

---
## 📂 Project Structure & Topics Covered

This repository is organized into specific learning modules:

### 📁 01. BASIC CSS

- **`inline-css.html`** – Using the `style` attribute directly inside HTML tags.
- **`internal-css.html`** – Writing CSS inside the `<style>` block within the `<head>`.
- **`external-html.html`** – Linking an external `.css` file (`style.css`).
- **`selectors.html`** – Practicing Universal, Element, Class, and ID selectors.
- **`units-of-css.html`** – Understanding Absolute (`px`) vs. Relative (`%`, `em`, `rem`) CSS units.

### 📁 02. BOX MODEL

- **`box-model.html`** – Mastering Content, Padding, Border, and Margin behavior.
- **`navbar.html`** – Building a navigation bar using the box model and alignments.
- **`practice_box_model.html`** – Practical exercises to reinforce box model concepts.

### 📁 03. PRACTICE

- **`relative_ele.html`** – Deep dive into CSS Positioning (`relative`, `absolute`) and parent-child structural behavior.

### 📁 assets

- **`style.css`** – The central external stylesheet used across the course projects.

---

## 💡 Key Takeaways

1. **Relative Elements:** Utilizing `position: relative` to offset elements smoothly from their normal document flow and establishing containment context for `absolute` positioned children.
2. **CSS Units:** Selecting the ideal unit for fluid layouts, leaning on `rem` for typography and `%` for column scaling.
3. **Nested Elements:** Organizing clean HTML parent-child hierarchies and understanding CSS value inheritance.
